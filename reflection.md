# Personal Reflection

Group: `Group C: Deadline knapp verPARSEd`

## What parts of the project did you spent most time on?
We spent most of our time planning and discussing the project. During our first meeting we only discussed the project we want to implement, what exactly the vulnerability should be, how to implement it and also deciding on a tech stack. The intended solution path was also already decided during the first meeting. One thing we decided later on was the obfuscation technique which was probably the most time consuming part besides the planning phase. The main issue was to decide on a difficulty level that would be appropriate while not making the challenge too difficult to solve. Our concern was that it would be either way too easy to solve or that we would make it too complex without realising. One factor was also that we wanted to design it so it would not only provide an appropriate difficulty level but would also provide an appropriate work load for the course.

## How did the group handle decision-making? Which channels (e.g., Mattermost, Meetings) were most effective or useful?
The decision making process within the group worked well overall. Most communication happened using Whatsapp for quick reminders and updates, while we met in person for larger design decisions and reviewing the project progress together. The meetings were especially useful when discussing our implementation ideas and deciding how the vulnerability and obfuscation techniques should work!

## Conflict & Resolution: Describe a moment where the group disagreed or faced a coordination "bottleneck." How was it handled? If this didn't occur, what do you think prevented this?
We did not face such a disagreement and we believe we were able to prevent that by planning the project thoroughly before implementing it. Since we already discussed the tech stack, idea and intended solution the implementation steps were very clear. Since we also had some time to think about the decisions after our first meeting, we could discuss possible ideas / differences before we started the implementation. Time or stress was also not an issue since we started the project early enough. Another factor that might have helped prevent conflicts is the fact that we have known each other before and have done previous group projects together already. 

## (Optional/Confidential) Is there anything regarding the workload distribution that the lecturers should be aware of?
The workload distribution was generally balanced and distributed fairly.
